Flexprice Checkout uses Razorpay to generate payment links and receive payment confirmations. Before creating your first Checkout Session, you need a connected Razorpay account and the correct webhook events configured in your Razorpay dashboard.

## Step 1: Connect Razorpay to Flexprice

If you haven't connected Razorpay yet, follow the [Razorpay Connection Setup guide](/integrations/razorpay/connection-setup) to add your API keys and webhook secret. Come back here once the connection is active.

## Step 2: Add the checkout webhook events in Razorpay

The base Razorpay connection requires `payment.captured` and `payment.failed`. Checkout Sessions need **three additional events** on top of those.

Go to your Razorpay Dashboard: **Settings → Webhooks → [your Flexprice endpoint]** and enable all five events below:

| Razorpay event | Why Checkout needs it |
|---------------|----------------------|
| `payment.captured` | Marks the Flexprice payment as `succeeded` and reconciles the invoice |
| `payment.failed` | Marks the Flexprice payment as `failed` |
| `payment_link.paid` | Triggers subscription activation and invoice finalization |
| `payment_link.cancelled` | Cleans up the draft subscription, invoice, and payment; marks session `failed` |
| `payment_link.expired` | Same cleanup as cancelled; marks session `expired` |

<Warning>
  If `payment_link.paid` is missing, Flexprice will never receive the payment confirmation and the subscription will stay in `draft` indefinitely. Always verify all five events are enabled.
</Warning>

### Your webhook URL

The Razorpay webhook endpoint for Flexprice follows this pattern:

```
https://api.cloud.flexprice.io/v1/webhooks/razorpay/<TENANT_ID>/<ENVIRONMENT_ID>
```

You can find your `TENANT_ID` and `ENVIRONMENT_ID` in **Settings → General** in the Flexprice dashboard.

## Step 3: Verify the webhook secret

Flexprice verifies the `X-Razorpay-Signature` header on every inbound webhook. If the webhook secret in your Razorpay dashboard doesn't match the one stored in your Flexprice connection, all events will be rejected silently.

To check: go to **Settings → Connections → Razorpay** in Flexprice and confirm the webhook secret matches what's shown in your Razorpay webhook endpoint settings.

## Step 4: Test the end-to-end flow

Use Razorpay test mode to verify everything is wired up before going live.

**Test card numbers (Indian payments):**

| Network | Card number | CVV | Expiry |
|---------|-------------|-----|--------|
| Visa | `4386 2894 0766 0153` | Any 3 digits | Any future date |
| Mastercard | `2305 3242 5784 8228` | Any 3 digits | Any future date |

**Test UPI IDs:**

| UPI ID | Simulates |
|--------|-----------|
| `success@razorpay` | Successful payment |
| `failure@razorpay` | Failed payment |

### What to verify

After completing a test payment, confirm all of these:

- Session status moves from `pending` to `completed`
- Subscription status moves from `draft` to `active`
- Invoice status moves from `draft` to `finalized`
- Your server receives the `checkout.session.completed` webhook event

If the session stays `pending` after payment, check the Razorpay webhook delivery log for failures and confirm all five events are enabled on the correct endpoint URL.

## How Flexprice processes each event

| Razorpay event received | What Flexprice does |
|------------------------|---------------------|
| `payment_link.paid` | Activates subscription, finalizes invoice, marks payment `succeeded`, sets session to `completed`, fires `checkout.session.completed` |
| `payment_link.cancelled` | Soft-deletes subscription + invoice + payment, sets session to `failed`, fires `checkout.session.failed` |
| `payment_link.expired` | Same cleanup, sets session to `expired`, fires `checkout.session.expired` |
| `payment.captured` | Updates payment record to `succeeded`; reconciles with invoice independently of payment link events |
| `payment.failed` | Updates payment record to `failed` |

Flexprice always returns HTTP 200 to Razorpay, even if processing fails internally. Errors are logged on the Flexprice side. This prevents Razorpay from retrying indefinitely on transient failures.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| Session stays `pending` after customer pays | `payment_link.paid` not enabled in Razorpay | Add the event in Razorpay Dashboard → Webhooks |
| Session goes `failed` immediately | Webhook secret mismatch | Re-copy the webhook secret from Razorpay into your Flexprice connection |
| Session creation returns an error | Razorpay connection inactive | Check connection status in Settings → Connections |
| Subscription remains `draft` | `payment_link.paid` webhook processed but event destination misconfigured | Verify the webhook URL includes the correct tenant and environment IDs |
