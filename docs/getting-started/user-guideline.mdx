## Why Features Matter

In FlexPrice, **Features define what your customers can use or consume**.  
Before creating plans or subscriptions, you must first create features.

Examples of features:

- API calls
- Number of users
- Storage usage
- Access to a premium module

---

## Step 1: Navigate to Features

After logging in to the FlexPrice dashboard:

1. Look at the **left sidebar**
2. Click on **Features**

You will be redirected to the **Features List page**, where all created features are shown.

---

## Step 2: Create a New Feature

On the Features List page:

1. Click the **Add Feature** button
2. You will be navigated to the **Feature Creation** page

Here, you need to fill in the following fields.

## Overview

This document explains the **subscription flow in Flexprice** from a **developer and user perspective**.  
It walks through each **core entity**, explains **why it exists**, and shows **how everything connects** to successfully start billing a customer.

This guide is useful if you are:

- New to Flexprice
- Building a billing module for your SaaS
- Trying to understand how customers, plans, and subscriptions work together

---

## Core Entities in Flexprice

Before creating a subscription, itâ€™s important to understand the main building blocks.

### 1. Customer

Represents **who you are billing**.

Examples:

- Individual user
- Company / Organization
- Team or workspace

A customer can have:

- One or more subscriptions
- Multiple invoices and payments
- Usage events (for metered billing)

---

### 2. Plan

Represents **what you are selling**.

A plan defines:

- Pricing (free, flat, or usage-based)
- Billing interval (monthly, yearly)
- Feature limits and entitlements
- Add-ons, taxes, and discounts

Plans are reusable and **not tied to a single customer**.

---

### 3. Subscription

Represents **the relationship between a customer and a plan**.

A subscription controls:

- When billing starts and ends
- Invoice generation
- Subscription status (active, trial, cancelled)
- Upgrades and downgrades
